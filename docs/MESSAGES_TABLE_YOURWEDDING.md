# YourWedding 전용 메시지 테이블 (동일 Supabase 프로젝트)

MyWedding의 `messages`와 **같은 프로젝트**를 쓰되, **테이블만 분리**합니다.

## 1. 환경 변수

`.env` / Vercel:

```env
VITE_SUPABASE_MESSAGES_TABLE=messages_your_wedding
```

테이블명은 Supabase에서 만든 이름과 **완전히 동일**해야 합니다.

## 2. SQL 예시 (`messages`와 동일 스키마)

Supabase → SQL Editor에서 실행 (테이블명만 맞게 수정):

```sql
create table if not exists public.messages_your_wedding (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now() not null,
  name text not null,
  relationship text not null,
  message text not null
);

alter table public.messages_your_wedding enable row level security;

create policy "read messages your wedding"
  on public.messages_your_wedding for select
  using (true);

create policy "insert messages your wedding"
  on public.messages_your_wedding for insert
  with check (true);
```

## 3. Realtime (새 메시지 즉시 반영)

1. **Database → Replication** 에서 `messages_your_wedding` 테이블을 **Realtime**에 추가  
   또는 프로젝트 설정에서 `supabase_realtime` publication에 테이블 추가.

## 4. 배포

`VITE_SUPABASE_MESSAGES_TABLE` 변경 후 **Vercel Redeploy** 필요.
